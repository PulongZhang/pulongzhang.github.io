# 文章写作规范

## 文件命名格式

文章文件必须放在 `_posts` 目录下，文件名格式为：

```
年-月-日-标题.md
```

示例：
- `2025-12-01-我的第一篇文章.md`
- `2025-12-15-学习笔记.md`

## 文章头部信息（Front Matter）

每篇文章开头必须包含以下格式：

```markdown
---
title: 文章标题
tags: 标签1 标签2
---
```

## 常用 Front Matter 字段

```markdown
---
layout: article             # 推荐，使用 article 布局
title: 文章标题              # 必填
key: 20251203-article-name  # 必填，文章唯一标识符（格式：年月日-主题）
tags: 标签1 标签2           # 可选，多个标签用空格分隔
author: 张蒲龙              # 可选
aside:                      # 可选，侧边栏配置
  toc: true                 # 显示文章目录
typora-root-url: ./..       # Typora 编辑器图片路径配置
---
```

**重要字段说明：**
- `key`: 文章的唯一标识符（**必填**）
  - **格式规范**：`YYYYMMDD-主题名称`（如 `20251203-my-first-post`）
  - 日期部分：8位数字，格式为年月日（如 `20251203`）
  - 主题部分：使用小写英文和连字符，简短描述文章主题（如 `ai-agent`、`life-thinking`）
  - 这个字段用于文章的内部引用和永久链接生成

## 作者管理（authors.yml）

如果文章需要指定作者，需要先在 `_data/authors.yml` 文件中添加作者信息。

### 添加新作者

在 `_data/authors.yml` 中按以下格式添加作者：

```yaml
作者名称:
  name: 作者名称
  avatar: 头像链接（可选）
  bio: 个人简介
  email: 邮箱地址（可选）
  github: GitHub用户名（可选）
  zhihu: 知乎ID（可选）
```

### 示例

```yaml
张蒲龙:
  name: 张蒲龙
  avatar:
  bio: 瞻彼日月,悠悠我思,道之云远,曷云能来
  email: puzzle1779746770@outlook.com
  github: PulongZhang
  zhihu: long-long-long-long-long-5

李子恒:
  name: 李子恒
  avatar:
  bio:
  email:
```

### 在文章中使用作者

在文章的 Front Matter 中使用 `author` 字段指定作者名称（必须与 authors.yml 中的键名完全一致）：

```markdown
---
layout: article
title: 文章标题
author: 张蒲龙
---
```

## 正文规范

### 一、段落规范

- 段落之间使用一个空行来分隔。
- 不使用首行缩进。这是标准的 Markdown 风格。段落通过与上一段的间距来区分，而不是缩进。

### 二、标点符号规范

**中英文标点区分：**
- 中文语境：全部使用全角标点符号。例如：`，` `。` `？` `！` `；` `：` `" "` `' '` `（ ）` `《 》`
- 英文语境：全部使用半角标点符号。例如：`,` `.` `?` `!` `;` `:` `"` `'` `( )`

**括号使用：**
- 括号内包含中文时，使用全角括号 `（）`
  - 示例：苹果公司发布了最新的人工智能（Apple Intelligence）。
- 括号内纯为英文或数字时，推荐使用半角括号 `( )`，且括号前后与中文字符之间应留一个半角空格
  - 示例：这项技术基于深度学习 (Deep Learning) 模型。

**书名号与引号：**
- 书名号 `《 》`：用于书籍、报刊、电影、艺术作品、法规文件等名称
- 引号 `" "`：用于引用、对话、表示特定含义或需要强调的词语
- 引号嵌套：外层用双引号 `" "`，内层用单引号 `' '`
  - 示例：老师说："你们要好好理解'天下兴亡，匹夫有责'这句话。"

**特殊标点：**
- 省略号：应使用标准的 `……`（占两个汉字空间），而不是六个点 `......` 或三个点 `...`
- 破折号：应使用标准的 `——`（占两个汉字空间），而不是两个减号 `--`

### 三、空格使用规范

**中英文之间加空格：**
- 在中文与英文单词/数字之间应增加一个半角空格
  - ✅ 正确：这个项目的 API 设计得非常优秀。请在 2024 年完成任务。
  - ❌ 错误：这个项目的API设计得非常优秀。请在2024年完成任务。

**数字与单位之间：**
- 不加空格：当数字与非字母单位（如 `%`）紧邻时
  - 示例：增长率达到 50%。
- 建议加空格：当数字与字母单位（如 km, GB, px）组合时
  - 示例：文件大小为 10 GB。

**标点与文字之间：**
- 全角标点：前后都不加空格
- 半角标点：标点后面加一个空格（除非是段落结尾），前面不加空格

### 四、数字、日期与单位规范

**数字使用：**
- 阿拉伯数字：用于公历纪年、计量、统计数据、编号等
  - 示例：第 12 章、87.5%、1998 年
- 汉字数字：用于非公历纪年（如 丙寅年）、词语中的数字（如 一石二鸟、星期五）、以及需要突出庄重感的场合

**日期和时间：**
- 推荐格式：2024 年 5 月 20 日

### 五、专有名词与术语规范

**大小写：**
- 遵循专有名词的官方或通用的大小写规范
  - 示例：CPU、iPhone、Google、JavaScript

**首次出现：**
- 对于不常见的英文缩写或专业术语，在文中首次出现时，建议给出中文全称或解释
  - 示例：我们采用了面向服务的架构 (Service-Oriented Architecture, SOA) 来构建系统。

**一致性：**
- 在整篇文档中，同一术语的翻译或写法必须保持统一

## 文章摘要分隔符

使用 `<!--more-->` 来分隔文章摘要和正文：
- 主页上只会显示 `<!--more-->` 之前的内容作为摘要
- 点击"阅读更多"后才能看到完整文章

## 完整示例

```markdown
---
layout: article
title: 我的第一篇文章
key: 20251201-my-first-post
tags: Life Draft
author: 张蒲龙
aside:
  toc: true
typora-root-url: ./..
---

这是文章的开头部分，会显示在主页摘要中。

<!--more-->

这是文章的正文部分，只有点击进入文章后才能看到。

## Markdown 语法支持

本博客支持完整的 Markdown 语法，包括：
- 标题（# ## ### 等）
- 列表（有序和无序）
- 链接和图片
- 代码块和行内代码
- 表格
- 引用块
- 数学公式（MathJax）
- 图表（mermaid, chartjs）
